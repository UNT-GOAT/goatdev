Auth Gate Pattern

Security
Every authenticated page MUST follow this pattern. The app starts hidden. No content renders until HerdAuth.requireAuth() confirms a valid session. If no session exists, the user is redirected to /login.html.
Rule: Never show page content before requireAuth() resolves. showApp(user) is the single entry point. This call must be the LAST line in every page's script. 
function showApp(user) {
  document.getElementById("app").style.display = "block";
}
// LAST line — gates everything
HerdAuth.requireAuth().then((user) => showApp(user));

HerdAuth API & Token Lifecycle

Auth
auth.js provides the HerdAuth global. All authenticated requests MUST use HerdAuth.fetch().
Methods:
• requireAuth() — Gate a page. Returns Promise<user> or redirects.
• getAccessToken() — Current JWT from sessionStorage (15 min).
• fetch(url, opts) — Drop-in for fetch(). Attaches Bearer, auto-refreshes, redirects if refresh fails.
• logout() — Clears tokens, revokes refresh server-side, redirects.
• getUser() — Cached {id, username, role}.
Token storage: Refresh tokens (30-day) in localStorage — XSS-extractable. Mitigated by rotation: each use invalidates old token. Stolen token = single use. 
Access token:  JWT (RS256 + kid), 15 min, sessionStorage
Refresh token: opaque, 30 days, localStorage, rotated on every use
Stream imgs:   auto-updated on refresh (img[src*="token="])

Camera Streams — Token-in-URL Auth

Security
<img> tags cannot set HTTP headers. JWT passed as ?token= in the URL.
1. Frontend appends ?token=<jwt> to img src
2. Caddy forward_auth → auth_verifier.py
3. Verifier extracts token from X-Forwarded-Uri header
4. JWT validated (RS256, expiry, kid)
5. On refresh (~14 min), auth.js auto-updates all img[src*="token="]
Tradeoff: Token in URL may be logged by proxies. Acceptable — 15 min lifetime. Same pattern as S3 signed URLs. 
["side", "top", "front"].forEach((cam) => {
  document.getElementById("stream-" + cam).src =
    CONFIG.API_BASE + "/api/viewfocus/stream/" + cam
    + "?token=" + HerdAuth.getAccessToken();
});

Role-Based UI

Pattern
Admin links injected dynamically based on user.role. Operators never see them.
Defense in depth: Client-side role checks are UX only. Server enforces require_admin on all /auth/users endpoints. 
if (user.role === "admin") {
  const link = document.createElement("a");
  link.href = "admin.html";
  link.textContent = "Admin";
  headerRight.insertBefore(link, navUser);
}

Service-to-Service Auth (Pi → EC2)

Architecture
Two independent auth layers:
• User-level: Browser → Bearer JWT → Caddy forward_auth
• Service-level: Pi → EC2 via X-API-Key (shared secret in .env)

EC2 middleware fails closed: no key configured = all non-health requests rejected.
Network: Pi ↔ EC2 via Tailscale (WireGuard encrypted). 
Browser                    Pi (Caddy)                EC2
  │── HerdAuth.fetch() ───→ │                         │
  │   Authorization: Bearer  │── forward_auth → OK    │
  │                          │── POST /analyze ──────→ │
  │                          │   X-API-Key: <secret>   │ → validate (fail-closed)
  │                          │←── grade result ───────│

Password Change — Current Password Verification

Feature
Available to ALL users. Verifies current password via login attempt before allowing change.
Important: Verify step uses raw fetch(), not HerdAuth.fetch(), to avoid interfering with the active session. Update step uses HerdAuth.fetch() for Bearer token.
Policy (client + server): 8+ chars, uppercase, lowercase, number, special character. 
// Step 1: Verify current password (raw fetch)
await fetch(CONFIG.AUTH_BASE + "/auth/login", {
  method: "POST", body: JSON.stringify({ username, password: currentPw })
});
// Step 2: Update password (HerdAuth.fetch for Bearer)
await HerdAuth.fetch(CONFIG.AUTH_BASE + "/auth/users/" + id, {
  method: "PUT", body: JSON.stringify({ password: newPw })
});

Full Architecture & Security Summary

Architecture
Complete request flow and security model. 
┌──────────────────────────────────────────────────────────────┐
│  BROWSER (CloudFront)                                        │
│  auth.js: access JWT (15min, sessionStorage)                 │
│           refresh token (30d, localStorage, rotated)         │
│  API calls:  HerdAuth.fetch() → Authorization: Bearer        │
│  Img tags:   src="...?token=<jwt>"                           │
└─────────────────────────┬────────────────────────────────────┘
                          │ HTTPS
┌─────────────────────────▼────────────────────────────────────┐
│  PI — Caddy (:8443)                                          │
│  forward_auth on EVERY /api/* → auth_verifier.py             │
│    Checks Authorization header, then ?token= fallback        │
│    On kid mismatch: refetches JWKS once, retries             │
│  /api/viewfocus/*  → camera proxy (:8080)                    │
│  /api/prod/*       → prod_server (:5000) → EC2 + X-API-Key  │
│  /auth/*           → proxied to EC2 auth (:8001)             │
└─────────────────────────┬────────────────────────────────────┘
                          │ Tailscale (WireGuard)
┌─────────────────────────▼────────────────────────────────────┐
│  EC2 — Docker                                                │
│  herdsync-auth (:8001)                                       │
│    Login rate-limited (5 fails/5min → 15min lockout)         │
│    Rate limit by IP AND username independently               │
│    Passwords: bcrypt, strong policy enforced                 │
│    JWTs: RS256 with kid for key rotation                     │
│    Refresh tokens rotated on use                             │
│    Deactivated users: immediate token rejection              │
│  ec2-api (:8000)                                             │
│    X-API-Key middleware (fail-closed)                         │
│    Health endpoints exempt                                   │
│  herdsync-db (:8002)                                         │
│    Animal/provider CRUD, grading results                     │
│    Unified serial_id sequence across species                 │
│  Postgres (RDS) — users, tokens, animals, sslmode=require    │
└──────────────────────────────────────────────────────────────┘

  ✓ Login rate limiting (IP + username, sliding window)
  ✓ Refresh tokens rotated on every use
  ✓ JWKS kid header for key rotation detection
  ✓ Pi↔EC2: Tailscale encrypted transport
  ✓ Passwords: strong policy, client + server
  ✓ Deactivated users: immediate token rejection
